Chrome Web Store正式上线

2010-12-07

北京时间昨天夜里，Google的Chrome浏览器网页应用商店正式上线，但目前仅支持英文版，如果在浏览器地址栏内加上中文参数zh-CN的话，Chrome Web Store会自动跳转到中文版的chrome浏览器扩展页面。

早在几天前就听说Google Chrome Web Store会在今天上线，于是我今天上网后首先就是查看关于Chrome Web Store的信息，然后就是自己试用一下。目前，我觉得Chrome Web Store中的Apps和网页应用没什么两样，很多应用只是网页版的简单的打包。而Google自己的Gtalk app却只能在Chrome OS系统上使用，在其他系统的Chrome浏览器上没法使用。这点我很无奈，因为我很期待Gtalk能够在浏览器最前端显示，就是看电影一样，可以在浏览器最前端显示一个小窗口，而不影响其他应用的显示。Chrome浏览器的Apps有四种打开方式可选，分别是以普通标签模式打开、以Pinned Tab模式打开、以窗口模式打开和以全屏模式打开。这些Web应用其实就是将网页打包一下，如果得到应用地址的话，很多应用都可以在其他浏览器下打开。不过，应该有一些应用可以支持离线模式，即断网之后照样可以使用。

Google自己的应用里面，我安装了Google Talk、Gmail、Google Docs、Google Calendar、Google Reader、Picasa等一系列Google自己的应用，但是其中的某些应用却不能正常使用。Picasa不能使用的原因是众所周知的，而Google Talk却是因为操作系统平台不行而不能使用，Google Reader是因为打开时是以非SSL连接方式，所以当阅读部分RSS源的时候会出现connect error错误，原因你懂的。

现在Chrome浏览器的扩展和应用已经不少了，虽然和Firefox浏览器比起来还差一大截，但是目前我所常用的扩展都已经有了，我之前很多次的试着让自己从Firefox浏览器转向到Chrome浏览器平台，但是Chrome浏览器总是让我在某些时候很不爽。Firefox浏览器的下个版本，也就是Firefox4.0的Beta版本UI太让我无奈了，难看到极点了，希望在Firefox4.0正式版时能在UI方面有一定的改进。而Chrome浏览器让失望的地方主要就是代理模式了，Chrome在WIN平台下没有自己的代理，而使用的是IE浏览器的代理，尽管Chrome浏览器已经有像Proxy Switchy这样优秀的扩展了，但是远远不如Firefox的autoproxy扩展给力。

不得不说的是，Chrome浏览器的界面是相当简单漂亮，很多地方设计的都是相当的棒。可以看提的出来，新版的Firefox和IE浏览器的界面都是受到了Chrome浏览器界面启发。不光是我，应该有很多人都希望Win下的Chrome浏览器能有一套自己的代理设置，而不是去使用IE的代理，这样对身处大陆的网民是非常有帮助的。还有一点就是，希望Chrome浏览器能够加上URL中的http://，这样的话，用户可以快速的进行SSL浏览方式的切换，现在我每次切换时，都得输入https://。
