Octopress博客第一篇内容

2012-07-07

使用 Wordpress 博客系统也有将近四年时间了，一直用着不错。前天晚上我突然想写博客，由于我的 Wordpress 博客一直放在国外，网速太慢，我实在忍受不了了，于是我就在我的 Github 上搭建了这个新的博客系统，速度挺快，编辑提交都很方便。

Octopress 博客系统的口号就是「A blogging framework for hackers」，单凭这个口号就知道这个博客系统很给力，我的这个博客系统是搭建在 Github Pages 上面。Ocotpress 博客系统是最近比较流行的静态博客系统，你可以写一篇 Markdown 格式的文本，然后 Octopress 会将这个 Markdown 文本通过一定的规则和模板生成一个静态的 HTML 页面。也就是说整个博客系统都是静态文件，不需要数据库、不需要动态语言、不需要备份、不怕攻击，从此告别对博客主机、系统、数据库的维护。

在Github上搭建Octopress

有两种方式可以在 Github 上搭建 Octopress 博客系统，分别是在 Github Pages 和 Github Project Pages 上搭建。下面我就以 Github Project Pages 为例来说一下搭建 Octopress 博客系统的步骤。

注： 我是将博客系统和博客系统源码放置在同一个 git 仓库的不同分支下，你也可以将博客系统和博客系统源码放置在不同的 git 仓库中。

在新机器上搭建Octopress写作环境

我是在 Github Project Pages 上面搭建的，系统环境为Ubuntu，下面以此为例来说明，假设博客源码在 master 分支，博客系统在 gh-pages 分支。

Markdown基本语法

Markdown 的目标是实现「易读易写」。简单来说，Markdown 是一个兼顾可读性与易用性的轻量级标记体系。Markdown 并不追求大而全，它只关心 HTML 里最常用的几个标记，对于一些不常用的标记它允许直接将 HTML 标记插入文本。

Markdown语法不算太难，记住几个关键的特殊字符用法就行了，用到什么查什么。关于Markdown语法的更多介绍可以参考以下内容：

  • Markdown语法快速入门 http://wowubuntu.com/markdown/basic.html

  • Markdown语法详细说明 http://wowubuntu.com/markdown/

其他说明

Octopress 主要是由 Git 和 Ruby 来驱动的，在 Windows 系统下使用的话可能比较折腾。我一直都使用 Ubuntu 系统，所以对我来说这要比在浏览器打开博客后台进行书写博客更加方便。
